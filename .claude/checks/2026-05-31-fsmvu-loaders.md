## Launch splash (PWA "click the icon" → login)
The native **OS splash** (the icon that "gets bigger" when you tap the installed PWA) is generated by the OS from the manifest icon + `background_color` and **cannot be animated** — it's a static PNG. So instead I added an **in-app one-shot splash** on `index.html` that picks up exactly where the OS splash leaves off:
- Full-screen overlay painted `#0f172a` (== manifest `background_color`) → seamless hand-off from the OS splash.
- The seal **scales up** (`.55→1.07→1`, deblur) — continuing the "icon grows" motion — holds, then the overlay fades to reveal the login (~1.6s total).
- `#fsmvuSplash` + tiny inline script (runs before Firebase): **tap to skip**, 2.2s safety timeout, and **fully skipped under `prefers-reduced-motion`** (CSS `display:none` + JS).
- `index.html` now links `css/fsmvu-loader.css`.
- Smoke: `.claude/splash-smoke.py` → PASS (braces 68/68, keyframes present, tags balanced).

### ⚠️ Stale-cache bug found & fixed (this is why a crimson seal appeared parked on the left of the login page)
The loader/splash CSS changed several times, but `CACHE_VERSION` was left at **v3** the whole time. The service worker is **cache-first for CSS within a version**, so returning browsers/PWAs kept serving the *old* `fsmvu-loader.css` — the copy that had `.fsmvu-seal` but **not** `.fsmvu-splash` / `--white` / the rotate masks. Result: the splash overlay lost `position:fixed` and the seal dropped into normal flow (left of the off-centre card), still crimson.
- **Verified** via local server + browser inspect that the *current* code is correct: `.fsmvu-splash` = `position:fixed`, seal `filter:brightness(0) invert(1)` (white), `animation-name:fsmvu-spin`, and the overlay self-removes (`display:none`) after ~2s.
- **Fix:** bumped `CACHE_VERSION` **v3 → v4** so `activate` purges the old `empower-shell-v3` cache and re-fetches fresh CSS/JS. Lesson: **bump `CACHE_VERSION` on every deploy that changes CSS/JS.**
- After deploy: hard-refresh (browser) or close+reopen the installed PWA — it may take one extra reload for the new service worker to take over.
