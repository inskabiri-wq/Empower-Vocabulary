# PWA — installable app + offline shell

**Date:** 2026-05-27
**Trigger:** User asked to add the app to phones (PWA). Free, no new services.

Turned Empower Lab into a **Progressive Web App**: installable to the
home screen, fast repeat-opens, opens offline — **without** ever
serving a stale version after a deploy, and **without** caching live
data or auth.

## The cache strategy (why it can't go stale)

This was the whole risk, given how often you deploy. Handled three ways:

1. **HTML = network-first.** Every page load fetches the latest from the
   network; the cache is only a fallback for when the device is offline.
   So a student can never get stuck on an old *page*.
2. **Static assets (css/js/png) = cache-first, but versioned.** Fast
   repeat-opens, and the cache name carries `CACHE_VERSION` ('v1'). On
   each meaningful deploy you bump it → the `activate` step purges the
   old cache → returning users pull fresh assets.
3. **The service worker file itself is served `no-cache`** (firebase.json),
   so the browser always re-checks it and notices a new version. Combined
   with `skipWaiting` + `clients.claim` + the one-time reload in
   pwa-register.js, a deploy reaches installed users cleanly.

**Cross-origin requests (Firebase, Google, gstatic, fonts, soundhelix)
are never intercepted** — they go straight to the network. So scores,
assignments, auth, and real-time data are NEVER cached and never stale.
(Verified by E2E: zero cross-origin entries in the cache.)

## ⚠️ Deploy habit (one line to remember)

When you deploy a change that touches CSS/JS, **bump `CACHE_VERSION` in
`y/service-worker.js`** (e.g. `'v1'` → `'v2'`). HTML is network-first so
pages are always fresh regardless, but bumping the version guarantees
returning users also get fresh static assets immediately. If you forget,
worst case is a returning user gets one-load-old JS until the next visit —
not broken, just slightly behind.

## How to install on a phone (after deploy)

**Android (Chrome):** open the site → tap the "Install" / "Add to Home
screen" banner (or ⋮ menu → Install app) → icon appears → opens fullscreen.

**iPhone (Safari only):** open the site in Safari → Share button → "Add to
Home Screen" → Add → icon appears.

Same web address students already use — the phone just wraps it as an app.

---

## How to fully remove the PWA (if ever needed)

1. Delete `manifest.json`, `service-worker.js`, `pwa-register.js`.
2. Remove the manifest `<link>` + apple meta + `<script src="/pwa-register.js">` from the 4 pages.
3. Remove the two firebase.json header blocks.
4. For already-installed users: the SW also accepts a kill message —
   `navigator.serviceWorker.controller.postMessage('UNREGISTER')` wipes
   its caches and unregisters.
